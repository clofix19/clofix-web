💡 How CloFix Load Balancer Works

CloFix Load Balancer sits between clients and backend servers, intelligently distributing traffic based on configurable algorithms. Health checks run continuously - unhealthy backends are automatically removed from rotation based on max_fails (default 3) and fail_timeout (default 30s).

Backend information hiding automatically strips 30+ sensitive headers from responses. Response body URL rewriting converts internal backend URLs to frontend domain in HTML, JSON, and XML content. Session persistence ensures users stay on the same backend via cookies.

All this runs with <1ms latency overhead using zero-copy forwarding and connection pooling.

❓ Frequently Asked Questions

What load balancing algorithms does CloFix support?

CloFix Load Balancer supports 5+ algorithms: Round Robin (sequential distribution), Least Connections (send to server with fewest active connections), IP Hash (client IP to same backend), Weighted (capacity-based distribution), and Random.

Does CloFix Load Balancer hide backend servers?

Yes. It automatically removes 30+ sensitive headers including Server, X-Powered-By, X-Backend-Server, X-Upstream, Via, and X-Real-IP from responses. It also rewrites URLs in HTML, JSON, and XML responses to show only the frontend domain.

What is the latency overhead?

<1ms average latency overhead with zero-copy forwarding and connection pooling. No measurable performance impact on your applications.

Does it support WebSockets and gRPC?

Yes, native WebSocket protocol upgrade with connection persistence across backend pools - perfect for real-time applications. Full gRPC support with HTTP/2 multiplexing and health checking protocol integration.

How does health checking work?

Active health checks run continuously with configurable intervals (default 10s), timeouts (default 3s), and expected status codes. Unhealthy backends are automatically removed from rotation. Passive health monitoring tracks real traffic failures (max_fails default 3, fail_timeout 30s).

Can I use CloFix Load Balancer for TCP/UDP traffic?

Yes, Layer 4 (TCP/UDP) load balancing is fully supported - perfect for databases, message queues, and legacy applications that don't use HTTP.
